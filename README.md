# I-AM Preparation Tool

A 12-domain NDIS support-needs preparation app. Capacitor (native iOS/Android shell) around the original I-AM web tool, with Supabase (Sydney) for accounts and cross-device sync.

**Privacy constraints baked into this project:** no analytics, no third-party tracking, Supabase Sydney region only, no participant data stored or processed outside Australia. Do not add any SDK or service that violates these.

## Project layout

```
www/                    The app itself (plain HTML/CSS/JS - no framework)
  index.html            Shell page + Content-Security-Policy
  css/app.css           All styling (green palette, rem-based sizes)
  js/data.js            The 12 domains, examples, barrier sheets (verbatim content - do not reword)
  js/config.js          <- PASTE YOUR SUPABASE URL + ANON KEY HERE
  js/state.js           App state, autosave, save-status events
  js/db.js              Encrypted on-device SQLite storage
  js/supabase.js        Supabase client, offline sync, account deletion call
  js/auth.js            Email sign-in / sign-up wrappers
  js/render/*.js        One file per screen
  js/main.js            Bootstrap + all button handlers (window.IAM)
  js/bundle.js          GENERATED by `npm run build` - never edit
  vendor/jspdf.umd.min.js  Bundled locally (no CDN at runtime)
supabase/
  migrations/0001_init.sql          Table + Row Level Security policies
  functions/delete-account/index.ts Account-deletion Edge Function
dev-server.js           Tiny static server for browser preview
capacitor.config.json   Native app config (app id: au.com.graceslowly.iam)
```

## 1. Prerequisites

- **Node.js** 20 or later (node.js.org, LTS installer)
- **For iOS builds:** a Mac with Xcode 15+ (App Store builds cannot be made on Windows - see note at the end)
- **For Android builds:** Android Studio (free, runs on Windows)
- A **Supabase** account (supabase.com, free tier is fine to start)

## 2. First-time setup

```bash
npm install
npm run build
```

## 3. Run it locally in a browser

```bash
npm run serve
```

Open http://localhost:8100. Until Supabase is configured the app runs in **local-only mode** (no sign-in screen, data stays in the browser) so you can test every screen immediately.

After you edit any file in `www/js/`, run `npm run build` again and refresh - or keep `npm run watch` running in a second terminal to rebuild automatically.

## 4. Supabase project setup (Sydney)

1. Sign in at supabase.com → **New project**.
2. **Region: Sydney (ap-southeast-2).** This is the step that keeps all participant data in Australia - double-check it before creating, the region cannot be changed later.
3. Pick a strong database password and store it in a password manager.
4. Once the project is ready, open **SQL Editor**, paste the whole contents of `supabase/migrations/0001_init.sql`, and run it. This creates the `assessments` table with Row Level Security so each signed-in user can only ever read or write their own row.
5. Go to **Project Settings → API**. Copy the **Project URL** and the **anon public** key into `www/js/config.js`. (Never put the `service_role` key anywhere in the app.)
6. Go to **Authentication → Providers → Email** and leave email+password enabled. Under **Authentication → URL Configuration** you can ignore redirect URLs for now (the app signs in directly, not via links).
7. Deploy the account-deletion function. Install the Supabase CLI (supabase.com/docs/guides/cli), then:
   ```bash
   supabase login
   supabase link --project-ref YOUR-PROJECT-REF
   supabase functions deploy delete-account
   ```
8. Rebuild the app (`npm run build`). Sign-up/sign-in now appears, and data syncs.

## 5. Run on a real device

### Android (works from this Windows machine)

```bash
npm run add:android     # first time only
npm run sync            # after every JS change
npm run open:android    # opens Android Studio
```

In Android Studio: pick your phone/tablet from the device dropdown (enable USB debugging on the device first: Settings → About → tap Build number 7 times → Developer options → USB debugging) and press Run.

### iOS (requires a Mac)

```bash
npm run add:ios         # first time only
npm run sync
npm run open:ios        # opens Xcode
```

In Xcode: select your team under Signing & Capabilities, plug in the iPad/iPhone, press Run.

## 6. Testing checklist before submitting anywhere

- [ ] Sign up, confirm email, sign in
- [ ] Complete a partial assessment, force-quit the app, reopen - it resumes where you left off
- [ ] Turn on aeroplane mode, keep editing - the save bar should say it will sync later; turn network back on - it syncs
- [ ] Sign in on a second device - the assessment appears
- [ ] Generate the PDF and plain-text report and share them via the native share sheet
- [ ] VoiceOver (iOS) / TalkBack (Android) pass: every screen reachable, role cards announce as buttons with pressed state, save status announced
- [ ] System text size set to largest: all screens remain usable
- [ ] Account screen → type DELETE → account and data gone; sign-in with the old password fails
- [ ] "Start fresh" clears the draft

## A note on building for iOS without a Mac

Apple requires Xcode on macOS to build and upload. Options, in order of simplicity:
1. Borrow/buy any Apple-silicon Mac (a used Mac mini is the cheapest path).
2. A cloud Mac service (e.g. MacStadium) - rent by the month when you need to build.
3. A CI service with macOS runners (more setup; not recommended for a solo owner at this stage).
